Dört Adımda Projeye Özel AI

1

Kaynak Eklenir

2

4

Ekip Kullanmaya Başlar

Soru sormak, kod review istemek, PR analizi yaptırmak, onboarding desteği almak — hepsi projeye özel bilgiyle, projeye özel modelle yanıtlanır.

Kullanım Senaryoları

Kurumsal Projelerde Archon Ne Sağlar?

NexaCore — çok modüllü, çok para birimli kurumsal fintech platformu · Java/Spring Boot · 600K+ satır kaynak kodu

Analiz
"Uluslararası transfer işlemlerinde çoklu para birimi kur farkı hesaplaması hangi servis katmanında yönetiliyor? FX oranları için harici API entegrasyonu mu var, yoksa dahili kur tablosu mu tutuluyor?"
Archon 340 sayfalık gereksinim dokümanını, 12 adet Architecture Decision Record'u (ADR) ve 3 sprint'lik analiz notunu çapraz sorgular. ExchangeRateService'in sorumluluğunu, birincil FX API + fallback strateji konfigürasyonunu ve mevzuata uyum notlarını tek yanıtta dokümante eder. Analist manuel okuma yerine 3 saati 90 saniyeye indirir.
Geliştirme
"Sipariş onay akışında eşzamanlı istek geldiğinde aynı sipariş iki kez onaylanabilir mi? Optimistic locking veya idempotency token uygulanmış mı?"
Archon OrderApprovalService kaynak kodunu ve ilgili entity'leri analiz eder: "@Version ile optimistic locking mevcut ancak idempotency token kontrolü eksik — OrderController.approve():134'te duplicate submission riski var. Benzer bir koruma PaymentService'te uygulanmış, aynı pattern buraya da uygulanabilir." Geliştirici doğru kaynağa, doğru satıra, karşılaştırmalı örnekle yönlendirilir.
Code Review
PR #512 açıldı — ödeme modülüne gece toplu mutabakat (batch reconciliation) servisi eklendi. 1.400 satır değişiklik.
Archon NexaCore'un mimari standartlarına ve güvenlik profiline göre inceleme yapar. Tespit edilen kritik bulgular PR comment olarak önceliklendirilmiş şekilde iletilir:
[Kritik] ReconciliationBatch.process() içinde yakalanan checked exception loglanmadan yutulmuş — mutabakat hatası sessiz kalır.
[Yüksek] @Transactional(rollbackFor = Exception.class) eksik — kısmi yazım başarısız olduğunda tutarsız veri kalabilir.
[Orta] Batch fetch size ayarlanmamış; 50.000+ kayıtta OutOfMemoryError riski. Aynı proje içinde ReportBatchService'te önerilen konfigürasyon mevcuttur.
Bakım
Production'da yüksek yük altında StackOverflowError alınıyor:
"Exception in thread 'async-4' java.lang.StackOverflowError at com.nexacore.reporting.ReportAggregator.aggregate(ReportAggregator.java:89)"
Hata aralıklı, yeniden üretilemiyor.
Archon kaynak kodu ve bakım notlarını birlikte analiz eder. ReportAggregator.aggregate() ile ReportNode.calculate() arasında belirli rapor şablonu kombinasyonlarında döngüsel çağrı zinciri oluştuğunu tespit eder. Sorunu yeniden üreten veri koşulu ve önerilen çözüm — referanslı kaynak satırlarıyla birlikte — tek yanıtta sunulur.
